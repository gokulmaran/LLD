Creational Design pattern
Singleton       ‚Üí Only one instance (global access)
Factory Method  ‚Üí Subclass decides object creation
Abstract Factory‚Üí Family of related objects
Builder         ‚Üí Step-by-step object creation
Prototype       ‚Üí Copy existing object (clone)

Structural Design pattern
Adapter    ‚Üí  Converts interface
Bridge     ‚Üí  Decouples abstraction & implementation
Composite  ‚Üí  Tree structure (whole-part)
Decorator  ‚Üí  Adds behavior dynamically
Facade     ‚Üí  Simplified interface to subsystem
Flyweight  ‚Üí  Shares common data
Proxy      ‚Üí  Controls access / lazy loading

Behavioral Design Patterns
Chain of Responsibility ‚Üí Pass request along chain
Command                ‚Üí Encapsulate actions as objects
Interpreter            ‚Üí Define grammar & interpreter
Iterator               ‚Üí Sequential access to collection
Mediator               ‚Üí Central controller for communication
Memento                ‚Üí Save & restore state
Observer               ‚Üí Notify dependents on change
State                  ‚Üí Change behavior by state
Strategy               ‚Üí Swap algorithms dynamically
Template Method        ‚Üí Define algorithm skeleton
Visitor                ‚Üí Add operations without changing classes
