Explanation

Problem definition

A parking lot is a designated area for parking vehicles and is a feature found in almost all populated areas. Parking lots are a necessity in cities and suburban areas for both residential and commercial purposes. The parking lot can have multiple floors with each floor having a different number of slots and different types of slots.

What to expect

A typical parking lot system has several components, each with specific constraints and requirements. The interviewer expects you to discuss:

  • Payment flexibility — multiple exit points, multiple payment methods (cash, credit, coupon)
  • Parking spot types — motorcycle, compact car, large vehicle, accessible
  • Multi-floor support — tracking paid customers per floor
  • Capacity display — showing available/occupied counts per spot type
  • Ticket system — entry time, duration calculation, fee structure

Scope

We will design a system supporting multiple floors, multiple entrances and exits, automatic ticket generation, and a flexible payment interface.

R1 Vehicle types
The parking lot must accommodate motorcycles, cars, vans/trucks. Each vehicle type maps to compatible spot types.
R2 Spot types
Four spot categories:
  • Motorcycle spots — only motorcycles
  • Compact spots — cars only
  • Large spots — vans, trucks, and cars
  • Accessible spots — reserved for disability vehicles
R3 Multi-floor support
The lot can have multiple floors. Each floor tracks its own available spots independently.
R4 Entry & exit panels
Multiple entrance panels issue tickets on arrival. Multiple exit panels process payment and release the spot.
R5 Payment flexibility
Customers can pay via cash, credit card, or coupon at any exit panel. Coupons provide a percentage discount.
R6 Fee calculation
Fees are time-based with a minimum 1-hour charge. Rates differ by vehicle type (motorcycle < car < van/truck).
R7 Singleton lot
Only one parking lot instance exists in the system (Singleton pattern). It manages all floors, panels, and active tickets.
R8 Capacity display
The system can report available spots per type per floor at any time.

Actors

  • Customer — drives in, receives ticket, parks, pays, exits
  • Parking Agent — can manually issue or validate tickets
  • System — auto-assigns spots, calculates fees, tracks occupancy

Use cases

UC1 Park vehicle
Customer arrives → system finds best available spot → issues ticket with entry time and spot info.
UC2 Pay and exit
Customer presents ticket at exit → system calculates fee → customer pays via chosen method → spot is freed.
UC3 Check availability
Display panel shows live count of available spots per type and per floor.
UC4 Apply coupon
Customer presents a coupon at exit — system applies discount before charging.

parking_spot.h — spot logic
#pragma once
#include <string>
#include <memory>
#include "vehicle.h"

class ParkingSpot {
private:
    std::string spotId;
    SpotType   spotType;
    SpotStatus status;
    int floor;
    std::shared_ptr<Vehicle> parkedVehicle;
public:
    ParkingSpot(const std::string& id, SpotType t, int f)
        : spotId(id), spotType(t), floor(f), status(SpotStatus::AVAILABLE) {}

    bool isAvailable() const { return status == SpotStatus::AVAILABLE; }

    bool canFit(VehicleType vt) const {
        switch (spotType) {
            case SpotType::MOTORCYCLE:  return vt == VehicleType::MOTORCYCLE;
            case SpotType::COMPACT:     return vt == VehicleType::CAR;
            case SpotType::LARGE:       return vt != VehicleType::MOTORCYCLE;
            case SpotType::ACCESSIBLE:  return vt == VehicleType::CAR;
            default: return false;
        }
    }

    void park(std::shared_ptr<Vehicle> v) {
        if (!isAvailable()) throw std::runtime_error("Spot occupied");
        parkedVehicle = v; status = SpotStatus::OCCUPIED;
    }
    void vacate() { parkedVehicle = nullptr; status = SpotStatus::AVAILABLE; }

    std::string getId()    const { return spotId; }
    SpotType    getType()  const { return spotType; }
    int         getFloor() const { return floor; }
};
ticket.h — parking ticket
#pragma once
#include <chrono>
#include <string>
#include <memory>
#include "parking_spot.h"

class ParkingTicket {
private:
    static int counter;
    std::string ticketId;
    std::shared_ptr<Vehicle>      vehicle;
    std::shared_ptr<ParkingSpot>  spot;
    std::chrono::system_clock::time_point entryTime;
    std::chrono::system_clock::time_point exitTime;
    bool paid = false;
    bool exited = false;
public:
    ParkingTicket(std::shared_ptr<Vehicle> v, std::shared_ptr<ParkingSpot> s)
        : vehicle(v), spot(s), entryTime(std::chrono::system_clock::now()) {
        ticketId = "TKT-" + std::to_string(++counter);
    }

    void markExit() { exitTime = std::chrono::system_clock::now(); exited = true; }
    void markPaid() { paid = true; }

    long long getDurationMinutes() const {
        auto end = exited ? exitTime : std::chrono::system_clock::now();
        return std::chrono::duration_cast<std::chrono::minutes>(end - entryTime).count();
    }

    std::string                   getId()      const { return ticketId; }
    std::shared_ptr<Vehicle>     getVehicle() const { return vehicle; }
    std::shared_ptr<ParkingSpot> getSpot()    const { return spot; }
    bool                          isPaid()     const { return paid; }
};
int ParkingTicket::counter = 1000;
payment.h — strategy pattern
#pragma once
#include <iostream>
#include <map>
#include "ticket.h"

class PaymentStrategy {
public:
    virtual bool processPayment(double amount) = 0;
    virtual ~PaymentStrategy() = default;
};

class CashPayment : public PaymentStrategy {
public:
    bool processPayment(double amount) override {
        std::cout << "  [Cash] Received: Rs." << amount << "\n";
        return true;
    }
};

class CreditCardPayment : public PaymentStrategy {
public:
    bool processPayment(double amount) override {
        std::cout << "  [Card] Charged: Rs." << amount << "\n";
        return true;
    }
};

class CouponPayment : public PaymentStrategy {
    double discountPct;
public:
    CouponPayment(double pct) : discountPct(pct) {}
    bool processPayment(double amount) override {
        double final = amount * (1.0 - discountPct / 100.0);
        std::cout << "  [Coupon] " << discountPct << "% off. Paid: Rs." << final << "\n";
        return true;
    }
};

class PaymentProcessor {
    std::map<VehicleType, double> ratePerHour = {
        { VehicleType::MOTORCYCLE, 20.0 },
        { VehicleType::CAR,        40.0 },
        { VehicleType::VAN,        80.0 },
        { VehicleType::TRUCK,      80.0 }
    };
public:
    double calculateFee(const ParkingTicket& ticket) {
        double hours = std::max(1.0, ticket.getDurationMinutes() / 60.0);
        return hours * ratePerHour.at(ticket.getVehicle()->getType());
    }
    bool pay(ParkingTicket& ticket, PaymentStrategy* strategy) {
        double fee = calculateFee(ticket);
        std::cout << "  Fee: Rs." << fee << "\n";
        bool ok = strategy->processPayment(fee);
        if (ok) ticket.markPaid();
        return ok;
    }
};
parking_floor.h — floor management
#pragma once
#include <vector>
#include <memory>
#include "parking_spot.h"

class ParkingFloor {
private:
    int floorNumber;
    std::vector<std::shared_ptr<ParkingSpot>> spots;
public:
    ParkingFloor(int n) : floorNumber(n) {}

    void addSpot(std::shared_ptr<ParkingSpot> spot) {
        spots.push_back(spot);
    }

    std::shared_ptr<ParkingSpot> findAvailableSpot(VehicleType vt) {
        for (auto& s : spots)
            if (s->isAvailable() && s->canFit(vt)) return s;
        return nullptr;
    }

    long countAvailable(SpotType t) const {
        long cnt = 0;
        for (const auto& s : spots)
            if (s->getType() == t && s->isAvailable()) ++cnt;
        return cnt;
    }

    int getFloorNumber() const { return floorNumber; }
    const std::vector<std::shared_ptr<ParkingSpot>>& getSpots() const { return spots; }
};
parking_lot.h — singleton orchestrator
#pragma once
#include <vector>
#include <unordered_map>
#include <memory>
#include "parking_floor.h"
#include "payment.h"

class ParkingLot {
private:
    static ParkingLot* instance;
    std::string name;
    std::vector<std::shared_ptr<ParkingFloor>> floors;
    std::unordered_map<std::string, std::shared_ptr<ParkingTicket>> activeTickets;
    PaymentProcessor paymentProcessor;

    ParkingLot(const std::string& n) : name(n) {}
public:
    static ParkingLot* getInstance(const std::string& n = "Default") {
        if (!instance) instance = new ParkingLot(n);
        return instance;
    }

    void addFloor(std::shared_ptr<ParkingFloor> f) { floors.push_back(f); }

    std::shared_ptr<ParkingTicket> parkVehicle(std::shared_ptr<Vehicle> v) {
        for (auto& floor : floors) {
            auto spot = floor->findAvailableSpot(v->getType());
            if (spot) {
                spot->park(v);
                auto ticket = std::make_shared<ParkingTicket>(v, spot);
                activeTickets[ticket->getId()] = ticket;
                std::cout << "[IN] Issued " << ticket->getId()
                          << " | Spot: " << spot->getId() << "\n";
                return ticket;
            }
        }
        std::cout << "[!] No spot for " << v->getPlate() << "\n";
        return nullptr;
    }

    bool exitVehicle(const std::string& ticketId, PaymentStrategy* strategy) {
        auto it = activeTickets.find(ticketId);
        if (it == activeTickets.end()) {
            std::cout << "[!] Ticket not found\n"; return false;
        }
        auto& ticket = it->second;
        ticket->markExit();
        std::cout << "[OUT] " << ticketId << "\n";
        bool ok = paymentProcessor.pay(*ticket, strategy);
        if (ok) {
            ticket->getSpot()->vacate();
            activeTickets.erase(it);
        }
        return ok;
    }

    void displayStatus() {
        std::cout << "\n=== " << name << " === Active: "
                  << activeTickets.size() << " tickets\n";
    }
};
ParkingLot* ParkingLot::instance = nullptr;
main.cpp — demo
#include "parking_lot.h"

int main() {
    auto* lot = ParkingLot::getInstance("City Centre Parking");

    // ── Build Floor 0 ──
    auto f0 = std::make_shared<ParkingFloor>(0);
    f0->addSpot(std::make_shared<ParkingSpot>("F0-M1", SpotType::MOTORCYCLE, 0));
    f0->addSpot(std::make_shared<ParkingSpot>("F0-C1", SpotType::COMPACT,    0));
    f0->addSpot(std::make_shared<ParkingSpot>("F0-L1", SpotType::LARGE,      0));
    f0->addSpot(std::make_shared<ParkingSpot>("F0-A1", SpotType::ACCESSIBLE, 0));
    lot->addFloor(f0);

    // ── Build Floor 1 ──
    auto f1 = std::make_shared<ParkingFloor>(1);
    f1->addSpot(std::make_shared<ParkingSpot>("F1-C1", SpotType::COMPACT, 1));
    f1->addSpot(std::make_shared<ParkingSpot>("F1-L1", SpotType::LARGE,   1));
    lot->addFloor(f1);

    // ── Vehicles arrive ──
    auto bike  = std::make_shared<Motorcycle>("TN01AB1234");
    auto car1  = std::make_shared<Car>("TN02CD5678");
    auto truck = std::make_shared<Truck>("TN03GH9012");

    auto t1 = lot->parkVehicle(bike);
    auto t2 = lot->parkVehicle(car1);
    auto t3 = lot->parkVehicle(truck);

    lot->displayStatus();

    // ── Vehicles exit ──
    CashPayment       cash;
    CreditCardPayment card;
    CouponPayment     coupon(20);

    if (t1) lot->exitVehicle(t1->getId(), &cash);
    if (t2) lot->exitVehicle(t2->getId(), &card);
    if (t3) lot->exitVehicle(t3->getId(), &coupon);

    lot->displayStatus();
    return 0;
}

File structure

The code is split into 7 header/source files, each mapping to one responsibility:

  • enums.h — all enum types (VehicleType, SpotType, etc.)
  • vehicle.h — abstract Vehicle + Motorcycle, Car, Van, Truck subclasses
  • parking_spot.h — spot state, canFit() logic, park/vacate
  • ticket.h — ticket with entry time and duration calculation
  • payment.h — Strategy pattern: Cash, CreditCard, Coupon + PaymentProcessor
  • parking_floor.h — per-floor spot management
  • parking_lot.h — Singleton orchestrator, main entry/exit flow
  • main.cpp — demo driver

To compile and run

All headers are in the same directory. Compile with:

g++ -std=c++17 main.cpp -o parking_lot && ./parking_lot

Key design decisions

  • Singleton — ParkingLot uses a static pointer; only one instance exists
  • Strategy — PaymentStrategy is a pure virtual base; swap implementations at runtime
  • shared_ptr — vehicle and spot ownership is shared across ticket and floor
  • No raw pointers — all heap objects managed via smart pointers except the Singleton

What we designed

A fully object-oriented Parking Lot system with multi-floor support, four spot types, flexible payment, and clean separation of concerns.

Patterns applied

  • Singleton — one global ParkingLot instance manages state
  • Strategy — PaymentStrategy decouples payment logic from the exit flow
  • Inheritance + Polymorphism — Vehicle hierarchy with virtual dispatch

Extension points
